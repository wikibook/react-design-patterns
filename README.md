이 저장소는 《모던 리액트 디자인 패턴》의 예제 코드 저장소입니다.
책의 예제 번호로 코드를 찾고, 독립 실행이 가능한 예제는 브라우저에서
확인할 수 있습니다. 브라우저 실행을 위해 원문에서 보완한 부분은 각 예제
카드와 실행 화면에 함께 표시합니다.

이 책은 리액트 애플리케이션에서 반복해서 마주치는 설계 문제에 이름을
붙이고, 패턴의 적용 배경과 트레이드오프를 함께 살펴봅니다. 특정 패턴을
정답처럼 적용하기보다 요구사항이 바뀌어도 유지보수하기 쉬운 구조를
판단하는 기준을 만드는 데 초점을 둡니다.

## 책의 구성

1. 상태 관리 패턴
2. 컴포넌트 구조 설계 패턴
3. UI 렌더링 및 표현 패턴
4. 로직 설계 패턴
5. 타입스크립트 패턴
6. 비동기 UI 패턴
7. 설계 및 아키텍처 패턴

## 이 저장소에서 확인할 수 있는 것

- 예제 번호와 제목으로 전체 예제 검색
- 장과 프로젝트 상태를 기준으로 예제 필터링
- 독립 실행 가능한 UI 예제의 브라우저 동작 확인
- 훅, 스토어, 타입, 중간 컴포넌트 등 참조용 소스 확인
- 브라우저 실행을 위해 원문에서 달라진 부분 확인

## 빠른 시작

### 실행 환경

- Node.js 22.12 이상 권장
- npm 10 이상

이 문서의 명령은 npm을 기준으로 설명합니다.

```bash
git clone https://github.com/wikibook/react-design-patterns.git
cd react-design-patterns
npm ci
npm run dev
```

터미널에 표시되는 로컬 주소를 열면 전체 예제 카탈로그가 나타납니다.
카탈로그에서 예제 번호나 제목을 검색하거나 장과 상태를 선택할 수 있습니다.

## 예제 번호와 경로 읽는 법

프로젝트에서도 책과 동일한 `n.n` 예제 번호를 사용합니다.

| 책의 예제 | 프로젝트 소스 | 실행 경로 |
| --- | --- | --- |
| 예제 1.1 | `src/pages/1/1.tsx` | `/1/1` |
| 예제 1.11 | `src/pages/1/11/index.tsx` | `/1/11` |
| 예제 6.48 | `src/pages/6/48.tsx` | `/6/48` |

하나의 파일로 표현할 수 있는 예제는 `<장>/<번호>.*`에 두고, 여러 파일이
필요한 예제는 `<장>/<번호>/index.*`를 진입점으로 사용합니다.

실행 경로는 독립된 화면에서 동작이나 결과를 확인할 수 있는 예제에만
제공합니다. 다른 예제가 참조하는 훅, 스토어, 타입, 중간 컴포넌트에 별도의
라우트를 만들면 책에 없는 래퍼나 임의의 입력값이 필요해지므로 소스 확인용으로
유지합니다.

## 예제 상태

| 상태 | 의미 | 독립 실행 경로 |
| --- | --- | --- |
| 실행 가능 | 단독 화면에서 동작이나 결과를 확인할 수 있는 예제 | 있음 |
| 소스만 있음 | 다른 예제가 참조하거나 독립 화면이 필요하지 않은 예제 | 없음 |
| 프로젝트 파일 없음(책에서만 제공) | 개념 코드, 별도 런타임, 외부 의존성 등의 이유로 이 프로젝트에 독립 파일을 두지 않은 예제 | 없음 |

`프로젝트 파일 없음`은 반드시 누락이나 오류를 뜻하지 않습니다. 카탈로그의 각
예제 카드에서 해당 상태로 분류한 이유를 확인할 수 있습니다.

## 원문 대비 변경

예제 코드는 책의 식별자와 내보내기 형태를 가능한 한 유지합니다. 실행 경로를
연결하기 위해 책에 없는 불필요한 기본 내보내기를 추가하지 않습니다.

브라우저 실행을 위해 Provider, 목 데이터, 환경별 shim 또는 최소한의 스타일이
필요할 수 있습니다. 이러한 보완이 있는 예제는 카탈로그와 실행 화면의
`원문 대비 변경` 영역에서 이유를 확인할 수 있습니다.

## 검증

전체 정적 검사와 프로덕션 빌드는 다음 명령으로 실행합니다.

```bash
npm run check
```

`npm run check`는 다음 작업을 순서대로 수행합니다.

1. ESLint 검사
2. 타입 검사
3. 프로덕션 빌드

책의 구조나 안티패턴을 그대로 보여주는 일부 예제에는 ESLint 경고가 남을 수
있으며, 실패 상황을 다루는 예제는 브라우저 콘솔 메시지를 발생시킬 수 있습니다.
`npm run check`는 정적 검사와 빌드만 수행하며 브라우저 상호작용까지 검증하지는
않습니다.

빌드 결과를 로컬에서 확인하려면 다음 명령을 사용합니다.

```bash
npm run build
npm run preview
```

## 디렉터리 구조

```text
src/
  pages/
    <장>/<번호>.*        # 한 파일로 구성된 예제
    <장>/<번호>/index.*  # 여러 파일로 구성된 예제
  ExampleCatalog.tsx     # 예제 검색 및 상태 카탈로그
  routes.tsx             # 독립 실행 가능한 예제 라우트
  shims/                 # 브라우저 실행을 위한 환경 호환 모듈
```
